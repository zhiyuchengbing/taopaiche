## 🚀 快速开始

### 方法一：使用快速开始脚本（推荐）

```bash
cd CLIP
python quick_start.py
```

这个脚本会自动：
1. 检查依赖和数据集
2. 引导你完成训练
3. 评估模型
4. 显示结果

### 方法二：手动步骤

#### 第一步：安装依赖

```bash
cd CLIP
pip install -r requirements.txt
```

**注意**：如果安装CLIP失败，请使用：
```bash
pip install ftfy regex tqdm
pip install git+https://github.com/openai/CLIP.git
```

#### 第二步：训练模型

```bash
python train.py
```

训练过程会：
- 自动从 `../output1` 加载数据
- 显示训练进度条
- 保存最佳模型到 `checkpoints/best_model.pth`
- 生成训练曲线到 `logs/loss_curve.png`

#### 第三步：评估模型

```bash
python evaluate.py
```

会输出：
- 准确率、精确率、召回率、F1分数
- 混淆矩阵
- 最优阈值建议
- 距离分布图

#### 第四步：使用模型预测

**命令行预测：**
```bash
python predict.py --img1 path/to/image1.jpg --img2 path/to/image2.jpg
```

**Python代码预测：**
```python
from predict import ImageSimilarityPredictor

# 创建预测器
predictor = ImageSimilarityPredictor()

# 预测两张图片是否同类
is_same, distance, confidence = predictor.predict(
    'image1.jpg',
    'image2.jpg',
    return_distance=True
)

print(f"是否同类: {is_same}")
print(f"距离: {distance:.4f}")
print(f"置信度: {confidence * 100:.2f}%")
```

## ⚙️ 配置参数

在 `config.py` 中可以修改以下参数：

### 训练参数
- `BATCH_SIZE = 32` - 批次大小（显存不足可以调小）
- `NUM_EPOCHS = 50` - 训练轮数
- `LEARNING_RATE = 1e-6` - 学习率
- `SAMPLES_PER_CLASS = 20` - 每个类别每轮采样的图片对数量

### 模型参数
- `CLIP_MODEL_NAME = "ViT-B/32"` - CLIP模型类型
  - 可选：`"ViT-B/32"`, `"ViT-B/16"`, `"ViT-L/14"`, `"RN50"`, `"RN101"`
  - ViT-B/32：速度快，显存占用少
  - ViT-L/14：效果好，但需要更多显存

### 损失参数
- `MARGIN = 0.5` - 对比损失的margin（同类距离应小于此值）

### 早停参数
- `EARLY_STOPPING_PATIENCE = 10` - 验证集不改善的最大轮数

## 🔧 常见问题

### Q1: 显存不足 (CUDA out of memory)

**解决方法：**
1. 减小批次大小：`BATCH_SIZE = 16` 或更小
2. 减少每类样本数：`SAMPLES_PER_CLASS = 10`
3. 使用更小的模型：`CLIP_MODEL_NAME = "RN50"`

### Q2: 训练速度太慢

**解决方法：**
1. 确保使用GPU：检查 `config.py` 中 `DEVICE = 'cuda'`
2. 增加数据加载线程：`NUM_WORKERS = 4` 或更大
3. 使用更小的模型：`CLIP_MODEL_NAME = "ViT-B/32"`

### Q3: 准确率不高

**解决方法：**
1. 增加训练轮数：`NUM_EPOCHS = 100`
2. 增加每类样本数：`SAMPLES_PER_CLASS = 50`
3. 调整margin参数：`MARGIN = 0.3` 或 `0.7`
4. 使用评估脚本找到的最优阈值
5. 使用更大的模型：`CLIP_MODEL_NAME = "ViT-B/16"`

### Q4: 如何使用自己的数据集

确保数据集按照以下格式组织：
```
你的数据集文件夹/
├── 类别1/
│   ├── image1.jpg
│   ├── image2.jpg
│   └── ...
├── 类别2/
│   ├── image1.jpg
│   └── ...
└── ...
```

然后修改 `config.py` 中的 `DATA_DIR` 指向你的数据集路径。

### Q5: 训练中断了怎么办

训练会自动保存检查点，可以修改 `train.py` 添加断点续训功能，或者直接重新训练（会自动使用最佳模型）。

## 🔍 预测结果解释

- **是否同类**：根据距离和阈值判断
- **欧氏距离**：特征向量之间的距离，越小越相似
- **置信度**：预测的可信程度（0-100%）
- **阈值**：判断同类/不同类的分界线（默认0.5）

## 📚 进阶使用

### 自定义阈值

```python
predictor = ImageSimilarityPredictor(threshold=0.3)  # 使用更严格的阈值
```

### 批量处理文件夹

```python
import os
from predict import ImageSimilarityPredictor

predictor = ImageSimilarityPredictor()

# 处理整个文件夹
folder = "path/to/folder"
images = [os.path.join(folder, f) for f in os.listdir(folder) if f.endswith('.jpg')]

# 比较所有图片对
for i in range(len(images)):
    for j in range(i+1, len(images)):
        is_same, distance, _ = predictor.predict(images[i], images[j], return_distance=True)
        if is_same:
            print(f"相似: {images[i]} <-> {images[j]} (距离: {distance:.4f})")
```

## 🎓 原理简介

本系统使用**孪生网络（Siamese Network）**架构：
1. 两张图片通过相同的CLIP编码器提取特征
2. 计算两个特征向量的欧氏距离
3. 使用对比损失训练，使同类图片距离小，不同类图片距离大
4. 预测时根据距离和阈值判断是否同类
